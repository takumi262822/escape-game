# 画面説明: Escape Game Portfolio

実装設計の詳細は DESIGN.md に分離し、この文書では難易度選択、探索画面、解除演出、クリア表示の見せ方に限定して説明する。

## 1. 目的
- 開発目的: 脱出ゲームの探索体験を、1 画面中心の UI で分かりやすく提示する。
- 評価してほしい点: タイトルから探索開始までの導線、視点移動の理解しやすさ、解除時の達成感。

## 2. 画面構成

### 2.1 タイトル・難易度選択
- 対象: index.html 初期表示
- 主な要素:
  - タイトル表示
  - 難易度選択ボタン
  - 導入メッセージ
- 役割: プレイヤーに最初の判断を一つだけ提示し、どの難易度から始めるかを即決できるようにする。

### 2.2 探索画面
- 対象: 難易度開始後のメイン表示
- 主な要素:
  - ルーム名ヘッダー
  - 中央の調査対象エリア
  - LEFT / RIGHT 移動ボタン
  - タイトルへ戻る導線
  - ナレーション表示
- 役割: どの部屋にいて、どの方向を見ているかを常に把握できるようにする。

### 2.3 暗号入力モーダル
- 対象: コード解除が必要な場面
- 主な要素:
  - コード入力欄
  - 解除ボタン
  - 閉じるボタン
- 役割: 探索画面を大きく崩さずに、謎解きの入力操作だけを切り出す。

### 2.4 インベントリとクリア表示
- 対象: 画面下部の所持品スロット、clear-overlay
- 主な要素:
  - アイテムスロット
  - 選択状態の表示
  - クリアメッセージ
- 役割: 取得物の蓄積と、クリア時の演出を同じ画面文脈の中で見せる。

## 3. 遷移と操作導線
- タイトルから難易度を選ぶと探索画面へ入る。
- 探索画面では LEFT / RIGHT で視点を切り替える。
- 調査対象やアイテムをクリックして進行フラグを更新する。
- 暗号入力が成功すると開放状態へ進み、条件達成後にクリア演出へ入る。
- クリア後はタイトル画面へ戻る。

## 4. 画面上の見どころ
- タイトルと探索画面を大きく切り分け、プレイヤーが今どの状態かを見失いにくくしている。
- ルーム名、方角、ナレーションの 3 点で現在地の理解を支える。
- インベントリを常時表示し、取得物が進行にどう関わるかを自然に意識させる。
- 入力モーダルは必要時のみ重ねて表示し、主画面の流れを壊しにくい構成にしている。
