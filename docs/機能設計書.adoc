== クラス設計

=== EscapeGameController（game-controller.js）

[cols="2,1,4", options="header"]
|===
|メソッド
|可視性
|処理内容

|constructor()
|public
|resetState() でゲーム状態を初期化。DOM 参照取得・Header 生成・init() 呼び出し

|resetState()
|public
|state オブジェクト（level/room/view/inventory/activeItem/flags）を初期値にリセット

|init()
|public
|難易度ボタン・移動ボタン・ロックボタン等のイベントを一括登録

|start(levelData)
|public
|Validator.isValidLevel() でデータ検証後、state を初期化してタイトル画面を隠し、ゲームを開始する

|changeRoom(key)
|public
|state.room を変更し view を 0 にリセット。ナレーターに移動メッセージを表示し render() を呼ぶ

|move(dir)
|public
|dir (+1 / -1) で VIEWS インデックスを更新し render() を呼ぶ。インデックスは 0〜3 でループする

|render()
|public
|現在 room と view に対応するオブジェクト一覧を UIComponents に渡して描画する

|getItem(key, icon)
|public
|key が inventory に未追加の場合のみ空きスロットを探してアイコンを配置し、state.inventory に key を追加する

|handleSlot(slot)
|public
|スロットの data-key を取得し、既に activeItem と一致する場合は inspect() を呼ぶ。そうでなければ選択状態に切り替えて activeItem を更新する

|inspect(key)
|public
|GameConstants.ITEM_HINTS[key] の説明文を say() に渡す。対応するヒントがない場合は固定文を表示する

|unlock()
|public
|ダイヤル入力値を現在 room の code と照合する。一致時は goalOpen/open フラグを立て toggleModal(false) 後に render()。不一致時は ERROR_MS 間 input-error スタイルを付与する

|toggleModal(visible)
|public
|引数が true の場合は is-active を付与してダイヤル入力 input にフォーカス。false の場合は is-active を除去する

|clear()
|public
|CLEAR_MESSAGES[level.id] をクリア画面に表示して is-active を付与する。RELOAD_MS 後に is-active を取り消し toTop() を呼ぶ

|toTop()
|public
|タイトル画面に戻り resetState() を実行する

|say(msg)
|public
|ナレーター要素にメッセージをセットする

|===

== ゲーム進行フロー

[source]
----
1. ページ読み込み → new EscapeGameController()
2. 難易度ボタン押下 → start(levelData)
3. 部屋レンダリング → render()
4. ユーザー操作
   a. 移動ボタン → move(±1) → render()
   b. オブジェクトクリック → handleInteraction()
   c. アイテムスロット → handleSlot()
   d. ダイヤルボタン → toggleModal(true)
   e. 解錠ボタン → unlock() → 正解なら changeRoom()
5. クリア条件成立 → checkClear() → クリア演出
6. RELOAD_MS 後 → toTop()
----

== セキュリティ考慮事項

* XSS: ナレーター・クリアメッセージへの動的テキスト出力は textContent を使用
* 入力検証: ダイヤル入力値は Validator で検証してから code 比較する
* ページ遷移: アンカーの href 遷移は行わず、ゲーム内状態遷移のみで完結する
