== 概要

本書は脱出ゲーム「THE LUXE ROOM」全体で使用する共通ユーティリティの仕様を記載します。 +
DOM 反映前は必ず XSSProtection を経由し、入力値は Validator で検証してから利用する。

---

== 1. XSSProtection クラス

=== 格納先

`src/utils/xss.js`

=== 説明

DOM への文字列反映前に HTML 特殊文字をエスケープし、XSS 攻撃を防ぐ。 +
Map オブジェクトを用いた変換テーブル方式で実装する。

=== 静的メソッド

==== sanitize(value)

.入力項目
[cols="2,2,4", options="header"]
|===
^.^|引数 ^.^|型 <.^|内容

<.^|`value` |any |任意の値。`null` / `undefined` は空文字に変換してから処理する。
|===

.出力項目
[cols="2,4", options="header"]
|===
^.^|型 <.^|内容

|string |HTML 特殊文字をエンティティに置換した文字列
|===

.変換表（Map による対応）
[cols="1,1", options="header"]
|===
|入力文字|変換後
|`&`|`&amp;`
|`<`|`&lt;`
|`>`|`&gt;`
|`"`|`&quot;`
|`'`|`&#x27;`
|===

== 2. Validator クラス

=== 格納先

`src/utils/validator.js`

=== 説明

ゲームレベル・ビューインデックス・コード入力値の形式チェックを行う。

=== 静的メソッド

==== isValidLevel(level)

.入力項目
[cols="2,2,4", options="header"]
|===
^.^|引数 ^.^|型 <.^|内容

<.^|`level` |any |検査対象のレベルオブジェクト
|===

.出力項目
[cols="2,4", options="header"]
|===
^.^|型 <.^|内容

|boolean |以下の全条件を満たせば `true`
|===

.判定条件
. `level` が truthy であること
. `level` が object 型であること
. `level.id` が string 型であること
. `level.code` が string 型であること

---

==== isValidViewIndex(index, viewsLength)

.入力項目
[cols="2,2,4", options="header"]
|===
^.^|引数 ^.^|型 <.^|内容

<.^|`index` |number |検査対象のビューインデックス
<.^|`viewsLength` |number |表示可能ビューの合計件数
|===

.出力項目
[cols="2,4", options="header"]
|===
^.^|型 <.^|内容

|boolean |以下の全条件を満たせば `true`
|===

.判定条件
. `Number.isInteger(index)` であること
. `index >= 0` であること
. `index < viewsLength` であること

---

==== isValidCodeInput(value)

.入力項目
[cols="2,2,4", options="header"]
|===
^.^|引数 ^.^|型 <.^|内容

<.^|`value` |any |検査対象の入力値
|===

.出力項目
[cols="2,4", options="header"]
|===
^.^|型 <.^|内容

|boolean |以下の全条件を満たせば `true`
|===

.判定条件
. `typeof value === 'string'` であること
. `value.trim().length > 0` であること（空文字・空白のみは不可）

== 4. 使用上の共通ルール

. 画面表示する全文字列は `XSSProtection.sanitize()` を通してから DOM に反映する
. コード入力の送信前に `Validator.isValidCodeInput()` で空文字を弾く
. localStorage は本プロジェクトでは使用しない（セッション内のみで状態を管理する）
