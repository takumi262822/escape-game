# コーディング規約

> プロジェクト: **escape-game（脱出ゲーム）**  
> 作成者: Takumi Harada  
> 作成日: 2026/04/06  

---

## 1. 概要

本ドキュメントは escape-game における JavaScript のコーディング規約を定めます。  
全ファイルはこの規約に沿って実装・レビューすることを原則とします。

---

## 2. フォルダ構成

```
src/
├── main.js                         # エントリーポイント（EscapeGameController を起動）
├── constants/
│   ├── code-definitions.js         # コード定義（難易度識別子・取得メソッド等）
│   └── game-constants.js           # ゲーム定数（VIEWS・ITEM_HINTS・CLEAR_MESSAGES等）
├── core/
│   ├── game-controller.js          # ゲーム進行全体の制御（部屋移動・謎解き判定）
│   └── main.js                     # コントローラーの初期化
├── levels/
│   ├── beginner-level.js           # 初級レベルデータ定義（LEVEL_BEGINNER）
│   ├── advanced-level.js           # 中級レベルデータ定義（LEVEL_ADVANCED）
│   └── elite-level.js              # 上級レベルデータ定義（LEVEL_ELITE）
├── styles/
│   └── style-manager.js            # CSS クラス操作ユーティリティ
├── ui/
│   ├── components.js               # オブジェクト・スロット等の DOM 生成
│   ├── header.js                   # ヘッダー（部屋名・視点表示）
│   └── footer.js                   # フッター HTML 生成
└── utils/
    ├── validator.js                # 入力コード・レベルデータのバリデーション
    └── xss.js                      # XSS サニタイズ関数
```

### 各ディレクトリの責務

| ディレクトリ | 責務 |
|---|---|
| `constants/` | 変更しない定数・コード定義のみを置く |
| `core/` | ゲーム進行制御の核。部屋移動・コード解錠・フラグ管理 |
| `levels/` | 各難易度の部屋・アイテム・フラグ定義のデータ。ロジックを含めない |
| `styles/` | CSS クラスの付け外しのみ。レイアウトロジックを含めない |
| `ui/` | DOM 生成・描画のみ。ゲームロジックを含めない |
| `utils/` | バリデーション・セキュリティなどの純粋関数 |

## 3. 命名規則

### 3-1. ファイル名

- **kebab-case** を使用する

```
game-controller.js   ✅
elite-level.js       ✅
gameController.js    ❌
```

### 3-2. クラス名

- **PascalCase** を使用する

```js
export class EscapeGameController { }   ✅
export class StyleManager { }           ✅
export class UIComponents { }           ✅
```

### 3-3. 関数・メソッド・変数名

- **camelCase** を使用する

```js
changeRoom(key) { }    ✅
toggleModal(v) { }     ✅
change_room(key) { }   ❌
```

### 3-4. 定数名

- **UPPER_SNAKE_CASE** を使用する
- ゲーム定数は `constants/game-constants.js`、コード定義は `constants/code-definitions.js` に集約する

```js
export const GameConstants = {
  VIEWS: ['front', 'right', 'back', 'left'],
  RELOAD_MS: 3000,
};
```

### 3-5. レベルデータ変数名

- レベルデータのエクスポート変数は `LEVEL_` プレフィックスを付ける

```js
export const LEVEL_BEGINNER = { ... };   ✅
export const LEVEL_ADVANCED = { ... };   ✅
export const LEVEL_ELITE    = { ... };   ✅
```

## 5. コメント規約

### 5-1. ファイルヘッダー

全ファイルの先頭に JSDoc ブロックコメントを記述する。

```js
/**
 * <ファイルの役割を一行で>
 * @author Takumi Harada
 * @date YYYY/MM/DD
 */
```

### 5-2. メソッド区切りコメント

メソッドの手前にセクションコメントを付ける。

```js
// --- 選択した難易度データでゲームを開始する ---
start(levelData) {
    ...
}
```

### 5-3. if 文コメント

**すべての `if` 文の直前行**に、何を判定しているかを日本語で説明するコメントを付ける。

```js
// レベルデータが無効な場合はエラーメッセージを表示して処理を中断する
if (!Validator.isValidLevel(levelData)) {
    this.say('準備中の難易度です。');
    return;
}
```

### 5-4. レベルデータ内の action コメント

レベルデータの `action` ・ `onAction` は連鎖する条件を段階的に説明する。

```js
action: (g) => {
    // 隠し通路が既に開通済みの場合はそのまま秘密の部屋に移動する
    if (g.state.flags.pathHall) return g.changeRoom('SECRET');
    // 手に持っているアイテムがバールの場合は連鎖条件を順に確認する
    if (g.state.activeItem === 'bar') {
        // 設計図をまだ読んでいない場合は延期メッセージを表示
        if (!g.state.flags.bluePrintRead) return g.say('...');
        ...
    }
}
```

## 7. レベルデータ設計

### 7-1. データ構造

各レベルは `views`（単一部屋）または `rooms`（複数部屋）どちらかで設計する。

```js
// 単一部屋（初級）
export const LEVEL_BEGINNER = {
    id: 'EASY', code: '742',
    views: [
        { render: (s) => '...', onAction: (g) => ... }
    ]
};

// 複数部屋（中級・上級）
export const LEVEL_ELITE = {
    id: 'HARD', code: '5856', startRoom: 'HALL',
    rooms: {
        HALL: { name: '中央ホール', views: [...] }
    }
};
```

### 7-2. action と onAction の使い分け

| プロパティ | 使用場所 |
|---|---|
| `action` | `rooms` 構造を持つレベル（中級・上級） |
| `onAction` | `views` 構造のみのレベル（初級） |

## 8. セキュリティ

### 8-1. XSS 対策

ナレーターへの表示は `XSSProtection.sanitize()` を通す。

```js
import { XSSProtection } from '../utils/xss.js';

// ✅ 正しい使用（say メソッド内で自動サニタイズ）
say(text) {
    this.dom.narrator.textContent = XSSProtection.sanitize(text);
}
```

### 8-2. コード入力のバリデーション

解錠コードは `Validator.isValidCodeInput()` で検証してから照合する。

## 9. 状態管理（フラグ設計）

### 9-1. state オブジェクト

ゲームの進行状態は `this.state` に集約する。直接プロパティを追加しない。

```js
this.state = {
    level: null,
    room: '',
    view: 0,
    inventory: new Set(),
    activeItem: null,
    flags: {}   // ←パズル進行フラグはここに追加
};
```

### 9-2. フラグの命名

フラグ名は `camelCase` で意味を明確にする。

```js
g.state.flags.pathHall     // ✅ 隠し通路の開通
g.state.flags.powerOn      // ✅ 電源復旧
g.state.flags.goalOpen     // ✅ ゴール解放
```
